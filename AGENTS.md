# AGENTS.md — bitrise-api

Generated Go client library for the Bitrise REST API.
Tech stack: Go 1.24, go-swagger code generation, GitHub Actions release automation.

Scope: applies to the entire repository. No subdirectory `AGENTS.md` files are needed here:
the repo is small, single-stack, and the generated `go/` tree does not have distinct local rules.

## Structure

```text
swagger.json                 Bitrise OpenAPI source downloaded from api-docs.bitrise.io
go/
  client/                    Generated HTTP client entry point and transport config
    operations/              Generated request parameter and response wrappers for API operations
  models/                    Generated API request/response models
.github/
  workflows/                 CLA enforcement and release drafter workflows
Makefile                     Download, validate, generate, build, and clean targets
catalog-info.yaml            Backstage API + component metadata
README.md                    Usage and development entry point
CONTRIBUTING.md              Contributor and CLA process
CLA.md                       Full contributor license agreement text
```

## Commands

| Action | Command |
|--------|---------|
| Show available targets | `make help` |
| Download latest Bitrise spec | `make download` |
| Validate `swagger.json` | `make validate` |
| Regenerate Go client | `make generate` |
| Build generated client | `make build` |
| Full refresh + build | `make all` |
| Remove generated code | `make clean` |

## Conventions

- Code under `go/` is generated by `go-swagger`; do **not** edit it manually.
- Generated files declare `// Code generated by go-swagger; DO NOT EDIT.` at the top.
- Regenerate the client with `make generate` after updating `swagger.json`.
- `make generate` runs `go run github.com/go-swagger/go-swagger/cmd/swagger@v0.30.5 generate client --with-flatten=minimal --skip-tag-packages`.
- `make build` is the compile check: `go build ./go/...`.
- This repo has **no repo-specific linter or formatter config** (`.golangci.yaml`, `golangci-lint`, `.editorconfig`, `mise.toml`, and `package.json` are absent). Do not document unenforced style rules.
- Ignore generated temp files listed in `.gitignore`: `/swagger_flat.json` and `/swagger_go.json`.

## CI / Release

- `.github/workflows/cla.yml` enforces CLA Assistant on pull requests and qualifying comments.
- `.github/workflows/release-drafter.yml` updates the GitHub release draft on pushes to `main`.
- `.github/dependabot.yml` updates Go modules and GitHub Actions weekly.
- There is no PR template in this repository.

## PR Process

- CODEOWNERS assigns all paths to `@Staffbase/south-park`.
- Contributors must satisfy the CLA process described in `CONTRIBUTING.md` and enforced by `.github/workflows/cla.yml`.
- Release notes are grouped by labels in `.github/release-drafter.yml`: `enhancement`, `bug`, `documentation`, `dependencies`.

## Domain Model

- `go/client/bitrise_api_client.go` exposes the main generated client and transport configuration.
- Default API target is `https://api.bitrise.io/v0.1`.
- `go/client/operations/operations_client.go` defines the generated `ClientService` interface for Bitrise API operations.
- `go/models/` contains generated DTOs for apps, builds, pipelines, organizations, addons, artifacts, secrets, credentials, and webhook payloads.
- This repository is a client library only: no local database, no service runtime, no application server.

## Configuration

- Runtime transport configuration is code-based via `client.TransportConfig` in `go/client/bitrise_api_client.go`.
- Override host, base path, or schemes with `WithHost`, `WithBasePath`, and `WithSchemes` when constructing a client.
- There is no repository-local `.env`, `docker-compose.yml`, or service config file.
